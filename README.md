## Обработка данных

Для более качественного прогноза модели мы комбинируем фичи, например: ```df_new['credit_to_income_ratio_12m'] = df_new['turn_cur_cr_avg_act_v2'] / df_new['salary_6to12m_avg'].replace(0, np.nan)```

Также особое внимание уделено категории профессия: например, для молодых людей с пропущенным значением в "профессии" мы предполагаем, что это может быть студент, также как и достаточно взрослый человек с пропущенным значением в профессии может быть пенсионер. Перед этим мы привели к стандартному виду все уникальные значения этого столбца и выделили группы, такие как "руководитель", "продавец", "учитель" и другие, где объединили схожие/одинаковые должности для создания полноценного категориального признака. 

Числовые признаки были отобраны с помощью корреляции, далее из них были отобраны 20 лучших по features_importance (для каждой модели набор признаков мог отличаться).

## Модель 

Модель устроена следующим образом: после предобработки фичей и логарифмирования таргета для работы с выбросами мы используем двухступенчатый подход. Логарифмирование таргета происходит из-за очень неравномерного распределения данных.

На первом этапе мы используем Catboost.Classifier для определения классов. Это необходимо для более успешной работы с "хвостами" наших данных. Следующим этапом для каждого из классов (их 2) мы строим отдельную модель CatboostRegressor с отдельным  обучением, подбором гиперпараметров и своими подобранными признаками. Для предсказания мы определниям, к какому классу относятся данные и после используем конкретную модель.
